How to use this lab guide

The files required to implement these labs can be downloaded as a zip file from http://github.com/instructormatters/html5css3

Alternatively, they can also be obtained by cloning the repository using Git.

Installing Git

Git can be downloaded for Windows and Mac from http://git-scm.com/downloads.

There is also a package for Ubuntu:

sudo apt-get install git git-gui gitk

Cloning the repository

Once Git is installed, run git clone:

git clone git://github.com/instructormatters/html5css3.git

This should create the html5css3/labs folder.

Structure

Each lab is divided into two folders, Before and After. Before provides with all files needed to start the exercise. After contains the finished version of the exercise to help you along.

Applications

The applications required for the labs can be found in the Appendix section at the end of this document.

Introduction

Tools Installation

Duration

20 minutes

Description

Download the labs folder as described in the Introduction.

Install the required tools for the labs. The installation and configuration instructions for each application can be found at the Appendix. Use them and install:

  • Node.js
  • Sublime Text Editor with the LESS package.

Once installed, execute the following inside the labs folder to install the tools that are required by the labs (no administrator privileges are required for this):

npm install

Now, open Sublime Text Editor and select Project → Open Project. Openlabs/labs.sublime-project and execute the following steps to get familiarized with the editor.

Manage files

  • Toggle sidebar on/off: Ctrl + K, Ctrl + B.
  • Open file: Ctrl+P
  • See several files at once: Shift + Alt + [1234589] (choose one number). Files can get aligned side by side, top vs bottom, etc. Once done, you can switch back with Shift+Alt+1.

Simple actions

  • Write any text, select a word, then press  Shift+Alt+W and type b.
  • Prepare a list of data, select all the text and hit Tab to indent it a bit. Now while holding the  Shift key, right click and drag  to select a rectangular selection of code. Type the start tag to get it inserted at the beginning of each line.

Multiple cursors

The previous step is just a concrete example of using multiple cursors:

  • Ctrl + click: create new cursor.
  • Ctrl + double click: create a new cursor selecting the word that was double-clicked.
  • Shift + click: select using the latest cursor (so: ctrl + click, then shift + click to add a selection to the set of cursors)
  • Any keyboard action: Operate on all cursors at once. Keyboard operations (like Ctrl+V) will work, and so will normal typing (like hitting End to write the ending tag at the end of each line).

To test this:

  • Write any text, ctrl+click at the beginning of each line. Write <p> or <li>
  • Ctrl + double click on several words,then type " (double quote)
  • Ctrl + double click on several words, hit  Shift + Alt + W, then type b

Classic HTML and CSS

Icon bar

Duration

30 minutes

Description

Design a button bar that shows the links as icons, without any text. In order to maintain accessibility and avoid the mystery meat, a title attribute has been included for each link.

  • Several icon sets are located in the img/ folder. The student must choose one of them to use during the lab.
  • Each icon set includes gray-scale and color icons. The latter shall be used for the hover effect of the links.
  • All icons must be arranged in a sprite, generated by http://www.spritefy.nl (Chrome and Firefox only).
  • The links should have float:left and margin-right:10px.
  • The text should be hidden the following CSS descriptor:
    a { 
  width: <icon width>; 
  height: <icon height>; 
  text-indent: 110%; 
  overflow: hidden; 
  white-space: nowrap; 
}

Compare the difference between this technique and the following two:

a {
  text-indent: -100000px;
  outline: none;
}
a {
  width: 0;
  padding: <height> <width> 0 0;
}

If in time, use a lossless compression tool to reduce the sprite size. Alternatives are PNGGauntlet (Windows), ImageOptim (Mac OS) or Trimage (Linux).

Browsers and SEO

IE Compatibility

Duration

30 minutes

Description

Correct the styles in the provided page to make them compatible with IE.

Gradient

The gradients are not visible in IE, showing instead the fallback background. To fix this, add to style.css the following rule to each selector with the appropriate colors (without line breaks):

/* IE6, IE7 */
filter: 
progid:DXImageTransform.Microsoft.gradient(startColorStr='#9c9c9c',EndColorStr='#7c7c7c');

/* IE8, IE9 */
-ms-filter:"progid:DXImageTransform.Microsoft.gradient(startColorStr='#9c9c9c',EndColorStr='#7c7c7c')";

After this change the first two buttons should display correctly, but the third button uses selectors that are not supported by IE < 9.

Third button

Create three different HTML pages, each one using a different technique:

  • CSS selectors for IE < 9, that style .button3 correctly.
  • Include ie7.js.
  • Use Google ChromeFrame. It will not be possible to test this code locally, due to a ChromeFrame bug. You will have to use a local server as described in the Appendix.
